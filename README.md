# Franke

Local-first заметочник с совместным редактированием в реальном времени без облачного хранения. Файлы живут локально у каждого участника, правки синхронизируются peer-to-peer через «слепой» relay, который не видит содержимого. Ключевая фишка: получатель ссылки редактирует заметку прямо в браузере, ничего не устанавливая.

Реализованы этапы 0–4 полностью и права доступа из этапа 5: live-редактирование с курсорами, вольт с `.md`-файлами на диске, E2E-шифрование и ссылки-приглашения, браузерный гость, комментарии, @упоминания, режим предложений, история версий, раздельные права чтения/записи с отзывом доступа. Подробный план и архитектура — в Obsidian-вольте (`Вайбкод/Franke`) и в `CLAUDE.md`.

## Установка

Готовые сборки — на [странице релизов](https://github.com/Escape64/Franke/releases/latest):

- **macOS** — `.dmg` (универсальный: Intel и Apple Silicon). Первый запуск: правый клик по приложению → «Открыть» — сборка пока не подписана сертификатом Apple.
- **Windows** — `…-setup.exe` (или `.msi`). Если SmartScreen предупреждает: «Подробнее» → «Выполнить в любом случае».
- **Linux** — `.AppImage` (`chmod +x` и запустить), `.deb` или `.rpm`.

Приложение проверяет обновления при запуске и предлагает установить новую версию само.

**Получили ссылку на заметку?** Устанавливать ничего не нужно — откройте её в браузере: редактор работает прямо на странице, правки сохраняются локально и синхронизируются с владельцем сквозь шифрование.

**Своё хранилище и Obsidian.** Вольтом может быть любая папка на диске (в т.ч. на внешнем накопителе) — выберите её через переключатель вольта в шапке сайдбара. Franke работает и с вольтами Obsidian напрямую: `.md` — обычные файлы, служебные данные Franke лежат в скрытой `.franke/`, папку `.obsidian/` приложение не трогает. Можно спокойно мигрировать или держать один вольт открытым и в Franke, и в Obsidian. Готовые `.md` можно просто перетащить из проводника в список — на папку или в корень.

## Структура

- `app/` — клиент: Vite + TypeScript, CodeMirror 6 + Yjs (`y-codemirror.next`), Tauri 2-оболочка в `app/src-tauri/`
- `relay/` — «слепой» WebSocket-relay (yjs 13 + y-protocols): пересылает только E2E-зашифрованные блобы, содержимого не видит

## Запуск (разработка)

```bash
npm install        # один раз, из корня
npm run relay      # relay на ws://localhost:1234
npm run dev        # веб-клиент на http://localhost:5173
```

Откройте http://localhost:5173 в двух окнах — увидите правки и курсоры друг друга.

Десктоп-окно (требует Rust):

```bash
cd app && npx tauri dev   # ожидает уже запущенный dev-сервер на 5173
```

## Свой relay одной командой (self-host)

Relay «слепой» — он лишь пересылает зашифрованные правки и не может прочитать заметки, поэтому его безопасно поднять на своём сервере:

```bash
docker compose up -d          # relay на ws://<ваш-хост>:1234
docker compose logs -f relay  # логи
docker compose down           # остановить
```

Порт можно сменить: `FRANKE_RELAY_PORT=4000 docker compose up -d`. Healthcheck — `GET /health`. Журнал комнат пишется на диск (volume `relay-data`) и переживает рестарт контейнера.

Публичный relay с TLS (wss) для VPS с доменом — отдельным compose-файлом: Caddy сам получает и продлевает сертификат Let's Encrypt:

```bash
# DNS: A-запись relay.example.com → IP сервера, затем:
DOMAIN=relay.example.com docker compose -f docker-compose.public.yml up -d
```

Чтобы клиент использовал ваш relay:

- при сборке: переменная `VITE_RELAY_URL=ws://<ваш-хост>:1234` (и `VITE_GUEST_URL_BASE=https://<адрес-веб-клиента>` для генерации ссылок);
- на лету в браузере: параметр `?relay=ws://<ваш-хост>:1234` в URL (запоминается в localStorage).

Ссылки-приглашения автоматически несут адрес relay в query-параметре, так что гость подключается к тому же relay, что и владелец. Адрес relay не секретен — секретны только ключи во фрагменте `#`, а они на сервер не попадают.
